Sumber data dari Laravel API:
- Base URL dari env VITE_API_BASE_URL
- Endpoint yang WAJIB dipakai:
  - GET /api/tv/{slug}
  - GET /api/tv/{slug}/absensi-hari-ini
  - GET /api/tv/{slug}/videos
  - GET /api/tv/{slug}/gallery
  - GET /api/tv/{slug}/realtime

Realtime Supabase (tanpa polling):
- Ambil konfigurasi realtime dari endpoint GET /api/tv/{slug}/realtime
- Jangan hardcode supabase url, anon key, channel, events
- Subscribe events sesuai konfigurasi dari backend
- Saat event masuk:
  - Debounce 300ms
  - Refresh absensi dari GET /api/tv/{slug}/absensi-hari-ini
  - Update Pinia store
- Jika event video/galeri nanti ditambahkan backend, siapkan hook refresh videos/gallery

Ketentuan performa:
- Gunakan virtualized list jika data absensi besar
- Hindari deep watch tidak perlu
- Gunakan computed/memoized transform untuk mapping status
- Batasi re-render komponen besar dengan pemecahan komponen yang jelas
- Unsubscribe Supabase channel saat unmount
- Reconnect strategy saat disconnected (backoff 1s, 2s, 4s, max 30s)

Contoh pseudo-code service realtime:

import { createClient } from '@supabase/supabase-js';

export async function connectRealtime(slug: string, onChange: () => void) {
  const cfg = await fetch(`/api/tv/${slug}/realtime`).then(r => r.json());
  const realtime = cfg.data;

  const supabase = createClient(realtime.url, realtime.anon_key);
  const channel = supabase.channel(realtime.channel || 'realtime_absensi_tv');

  for (const evt of realtime.events) {
    channel.on('postgres_changes', {
      event: evt.event,
      schema: evt.schema,
      table: evt.table,
    }, () => onChange());
  }

  channel.subscribe((status) => {
    console.log('Realtime status:', status);
  });

  return () => {
    supabase.removeChannel(channel);
  };
}
